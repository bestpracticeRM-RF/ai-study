Домашняя MLOps-платформа на k3s (single-node, RTX 5070 Ti). **Фазы 1-3 завершены**,
текущая — **Фаза 4: теория к собеседованиям + углубление в инструменты**.

- Актуальный план и статус: `plan-AI.md` (единственный источник правды по фазам)

Развёрнуто: Argo CD, MLflow, Airflow, Triton, PostgreSQL, MinIO, Vault+ESO,
VictoriaMetrics+Grafana, Evidently, JupyterHub, GitLab CE + runner, DVC,

## Формат занятий (Фаза 4)

Тема → 30-60 мин теории с вопросами «как на собесе» → практика на живом кластере →
конспект в `docs/theory/`. Использовать реальные инциденты платформы как кейсы
(CrashLoop, OOM, VM-operator краши, MLflow Host-403 — всё это было вживую).

## Технические правила

- CI/CD — только GitLab CI (self-hosted GitLab CE в кластере). Не предлагать GitHub Actions.
- Ingress: базово nginx. Gateway API — только с обоснованием под сложный ML-трафик
  (canary, header-based routing, gRPC, shadow). Без Istio/Linkerd без критичной нужды.
- Не предлагать SaaS без локальной/on-prem альтернативы. Приоритет — open-source.
- Учитывать 152-ФЗ, изоляцию контуров, MLSecOps (cosign, ESO, аудит) где уместно.
- Указывать версии инструментов/K8s, когда это влияет на совместимость.
- Спорные выборы (KServe vs Seldon, MLflow vs ClearML) — краткое сравнение + финальный
  выбор под этот стек и рынок РФ.
- Никогда не генерировать и не коммитить секреты/токены. Паттерн: Vault + ESO.
